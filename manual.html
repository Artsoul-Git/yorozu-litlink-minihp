初回セットアップ手順

  • 1
    GitHub Pages を有効にする
    リポジトリの Settings → Pages → Source: Deploy from a branch → main / (root) に設定。数分後 URL が発行される。
  • 2
    manual.html を .gitignore に確認する
    .gitignore に 06_セミナー事業部/outputs/Lit.linkでミニホームページを作ろう！/07_HTML版_v2/manual.html が記載されていること。このファイルは絶対に push しない。
  • 3
    admin.html でパスワードを設定する
    GitHub Pages の admin.html を開き、上の「初期パスワード」を入力して「設定・変更」をクリック。その後、好みのパスワードに変更することを推奨。
  • 4
    presenter.html の動作確認
    パスワードを入力してログインできることを確認する。
  • 5
    viewer.html の URL を控える
    admin.html に表示される「受講者向け共有URL」をコピーしておく。

セミナー当日の手順

  • 1
    受講者に viewer.html の URL を共有する
    チャット・メール・QRコードなど。受講者はスライドを閲覧できるが、エクスポートやプレゼンター機能は使えない。
  • 2
    presenter.html を自分のデバイスで開く
    パスワードを入力してログイン。タイマー・トークスクリプト・目次が使える。スライドは BroadcastChannel で viewer.html と同期する（同じデバイスで両方開いている場合）。
  • 3
    プロジェクターには viewer.html を表示する
    別ウィンドウで viewer.html を開き、プロジェクター側のディスプレイに移動。presenter.html の「全画面」ボタンで viewer.html を全画面にできる。

セキュリティについて

パスワードの保存場所

admin.html で設定したパスワードは、このブラウザのローカルストレージにのみ保存されます。GitHubのサーバーやリポジトリには一切保存されず、他のユーザーには見えません。

presenter.html と admin.html は同じデバイス・同じブラウザで使う必要があります。

このシステムのセキュリティレベル

  • 「URLを知らないと入れない」レベルのアクセス制限
  • パスワードのソースコードへの埋め込みなし（修正済み）
  • 技術的な知識があればJavaScriptを迂回できる — ただしセミナー資料の共有には十分
  • 本格的な認証が必要な場合はサーバーサイドの仕組みが必要
重要： presenter.html のパスワードを変更した場合、新しいデバイスで発表する際は そのデバイスの admin.html で再設定が必要です。ローカルストレージはデバイス・ブラウザごとに独立しています。
